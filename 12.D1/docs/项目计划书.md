一、项目概述
本项目以 ESP32-S3 为主控，搭载 MPU6050（加速度计+陀螺仪）、HMC5883L（三轴磁力计），实现可穿戴人体日常活动模式识别。
完整工程链路：传感器数据采集 → 传感器标定 → 信号预处理 → 时域/频域特征提取 → 特征选择与降维 → 统计分类器建模 → 模型评估 → 边缘部署演示。
项目目标：区分多种人体日常活动，完成完整模式识别工程闭环，验证传感器标定对识别精度的提升效果，并完成 PCB 扩展板设计（进阶要求）。
二、识别类别定义
本次识别共划分 6 类基础人体活动：
1.	静坐
2.	站立
3.	平地步行
4.	跑步
5.	上楼
6.	下楼
三、硬件平台
•	主控：ESP32-S3 开发板
•	传感器：MPU6050（六轴惯性传感器）、HMC5883L（三轴磁力计）
•	辅助配件：杜邦线、面包板、绑带、USB 数据线、移动电源
四、软件环境
1.	算法环境：Python 3.10 + NumPy + Pandas + Scikit-learn + Matplotlib
2.	嵌入式开发：Thonny (MicroPython)
3.	PCB 设计：KiCad 8.0
4.	版本管理：Git + Gitee
五、整体技术路线
1.	数据层：制定采集协议，多被试采集标定前/后两组数据集
2.	标定层：加速度计六位置标定、磁力计椭球拟合、陀螺仪零偏校正
3.	信号层：滤波、去重力、滑动窗口分割、标准化
4.	特征层：提取时域、频域特征，完成特征选择与降维
5.	建模层：贝叶斯、线性判别、SVM、随机森林等多分类器训练与调参
6.	评估层：LOSO留一被试交叉验证、混淆矩阵、F1、显著性检验
7.	部署层：模型轻量化，部署至 ESP32-S3 实现实时识别演示
六、风险与难点预判
1.	数据质量风险：传感器零偏、磁场干扰导致原始数据误差大，依赖标定优化
2.	数据泄漏风险：严格按被试划分训练集/测试集，禁止混分
3.	分类难点：上楼、下楼特征相似度高，易出现类别混淆
4.	硬件风险：I2C 通信异常、磁力计受 ESP32 天线磁场干扰
5.	PCB 风险：D3 中午前必须完成下单，否则无法参与优秀评级
